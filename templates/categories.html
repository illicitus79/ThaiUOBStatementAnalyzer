Spending Categories

Manage how transactions are automatically categorized. Each keyword is matched as a substring against the transaction description (case-insensitive). Use * as a wildcard — e.g. GRAB*FOOD matches "GRAB EXPRESS FOOD". When a keyword could match multiple categories, the category listed higher in the list wins. Transactions that match no keyword fall into Other.

Keyword matching rules:
  • Substring: GRAB matches "WWW.GRAB.COM", "GRAB FOOD", "GRABFOOD", etc.
  • Wildcard *: GRAB*FOOD matches any text between "GRAB" and "FOOD".
  • Priority: categories are checked top-to-bottom; the first match wins. More specific keywords should come first.
  • Conflicts: the same keyword cannot be in two categories — add it to the one that should take priority and the UI will warn you of conflicts.
